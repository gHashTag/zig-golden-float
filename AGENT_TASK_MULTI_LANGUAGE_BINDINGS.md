```markdown
# GoldenFloat Language Bindings Specification

## C-ABI Surface (Canonical Source)

### Level 1: Core Operations
- `gf16_from_f32(x: f32) -> gf16_t`
- `gf16_to_f32(g: gf16_t) -> f32`
- `gf16_add/sub/mul/div(a, b) -> gf16_t`
- `gf16_neg/abs(g) -> gf16_t`

### Level 2: Extended Operations
- `gf16_min/max(a, b) -> gf16_t`
- `gf16_fma(a, b, c) -> gf16_t`
- `gf16_copysign(target, source) -> gf16_t`

### Level 3: Predicates
- `gf16_is_nan/inf/zero/negative(g) -> bool`

### Level 4: φ-Math
- `gf16_phi_quantize(x: f32) -> gf16_t`
- `gf16_phi_dequantize(g: gf16_t) -> f32`
- `goldenfloat_phi() -> f64`
- `goldenfloat_trinity() -> f64`

## Binding Rules

1. **No native math**: All bindings MUST call C-ABI, never reimplement algorithms
2. **Conformance required**: Every binding MUST pass `conformance/vectors.json` tests
3. **Error handling**: Follow language idioms (exceptions in C++, Result in Rust, etc.)
4. **Constants**: Expose GF16_ZERO, GF16_ONE, GF16_PINF, GF16_NINF, GF16_NAN
5. **φ-math**: Expose `phi()`, `phi_sq()`, `phi_inv_sq()`, `trinity()`

## Required API Surface

Every language binding MUST implement:
- Conversions: `from_f32`, `to_f32`
- Arithmetic: `+`, `-`, `*`, `/`, unary `-`
- Predicates: `is_nan`, `is_inf`, `is_zero`, `is_negative`
- Constants: `zero`, `one`, `p_inf`, `n_inf`, `nan`
- φ-Math: `phi_quantize`, `phi_dequantize`, `phi`, `trinity`
```

**Match modes explained:**
- `"roundtrip"`: Convert f32 → GF16 → f32, check value preserved within tolerance

- `"is_nan"` / `"is_inf"` / predicate: Check predicate returns true
- `"predicate"`: Generic predicate check with expected boolean result

## Value Proposition (for README)

### Why GF16 Ecosystem Matters

- **Beyond FP16/BF16 trade‑offs.** GF16 (6 exponent / 9 mantissa bits) sits in the "golden corridor" between range and precision, offering FP32‑level accuracy on benchmarks like MNIST while enabling lower memory and higher throughput similar to FP16/BF16/FP8 formats.

- **One numeric core, many languages.** GoldenFloat exposes a stable C‑ABI with bindings for Zig, C, Rust, Python, C++ and Go, so the same arithmetic and φ‑math are available across research code, production runtimes and systems tooling.

- **Reproducible and verifiable.** A shared conformance test set (golden vectors) and cross‑language tests ensure bit‑wise consistent behavior across all bindings, reducing the typical drift and subtle bugs that plague multi‑language ML stacks.

- **Hardware‑ready by design.** GF16 is co‑developed with FPGA/ASIC pipelines: the same format used in software is already mapped to low‑resource arithmetic units, enabling realistic studies of accuracy vs. cost in custom accelerators.
